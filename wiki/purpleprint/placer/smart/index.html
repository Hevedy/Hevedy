    Smart Placer

    The Smart Placer is the dynamic containers for the Purpleprint Placer plugin can contain data and generate it with random placement and almost procedural systems allowing to spawn even sub actors of the same type and decorators around the instances, everything is properly calibrated and stored as instance for performance.

    Actors:

    • Smart Placer: This is the parent class for the smart placers, contains most of the code, only use this if you know what you are doing.
    • Smart Field: Can generate big amoung of instances in different options, like grids, points, random location, using Niagara, as scatter using a static mesh and others, supported by a big list of modificators and offset tools for random offsets and rotations, perfect for more structured but varied asset placement. Can be spawned via the browser.
    • Smart Spline: Generate instances across splines, with almost the same settings from Field, plus extra elements to control the spline transform and rotations in many different ways, including noise placers and more tool. Perfect for details across roads, floors, wall details, floating elements in space and many others. Can be spawned via the browser.

    Assets:

    • Smart Instance: Structure allows to define traces, complete transforms and all the settings for normal actors, culling, light filtering, shadows, and almost all the elements you find by default in the engine.
    • Smart DataAsset: Container that can generate instances based on a list of assets you provide, can be used as prefab for you actors, if you drag & drop it to the world, starting the 2.0 will create by default a Smart Placer.

    Features:

    • Customize all the settings you have or almost all the settings for light, shadows, culling from normal actors.
    • Can be used for static meshes, skeletal meshes, decals, particles, niagara particles and actors you want to, included sub actors of Smart Placer.
    • Allows to shift the origin of the components whatever you want, per component and per instance and globally.
    • Complete suite for random placement and almost procedural systems allowing to spawn even sub actors of the same type and decorators around the instances, everything is properly calibrated and stored as instance for performance.
    • Different alignment traces so you can align the actors to surfaces of any kind, walls, floors, ceilings...
    • Options of you taste to define areas, shape them as you wish with hollow and side options.
    • Bias and masking per axis for fine control of spawn.

    Instance System

    • Instances: List of dynamic instance definitions used for placement.
    • DefaultInstance: Fallback or template instance definition.
    • DataAsset: External data-driven asset to generate instances automatically.

    Instance Configuration

    Instance

    SmartInstanceStruct, defines the template and behavior of a single instance.

    Property Description
    Asset Main object (mesh, particle, etc.)
    StaticMeshType Defines mesh instancing method: Component, Instance, Hierarchical.
    Materials Overrides for material assignment.
    Parameter Optional parameter for Niagara/particles.
    NumDecorators, Decorators Children/sub-instances attached to this one.
    Transform Struct controlling instance transform rules.
    Physics Struct controlling collision, physics, mobility.
    Rendering Struct controlling visibility, LOD, shadowing, etc.
    InstancesNum Runtime number of generated instances.

    Transform

    InstanceTransformStruct, defines the global rendering parameters for light, shadows...

    Property Category Description
    TransformComponent Transform Global transform applied to all instances.
    TransformInstance Transform Per-instance transform offset.
    Origin Transform Reference point for bounding calculations.
    Extend Transform Optional override for bounding box extents.
    RandomLocation Transform\Random Min/Max range for random location.
    bUseLocationList Transform\Random Whether to use predefined location list.
    LocationList Transform\Random List of predefined random locations.
    RandomRotation Transform\Random Min/Max rotation (Pitch, Yaw, Roll).
    bUseRotationList Transform\Random Use predefined rotations instead of random.
    RotationList Transform\Random List of fixed rotation values.
    RandomScale Transform\Random Min/Max uniform or non-uniform scale.
    bRandomScaleUniform Transform\Random If true, uses X for uniform scale.
    bUseScaleList Transform\Random Use predefined scales instead of random.
    ScaleList Transform\Random List of specific scale vectors.
    TargetActor Transform\Target Actor to face/look at.
    bTargetActorInvertTransform Transform\Target Invert the look-at direction.
    TargetRotationMask Transform\Target Per-axis rotation mask.
    Trace Transform\Trace Trace alignment settings (surface detection, etc.).

    Trace

    TraceAlignStruct, defined the tracing options to align the instance to the world.

    Property Category Description
    Origin Trace Starting point of the trace. Only used when Direction == eCustom.
    Length Trace Trace distance. Ignored when Direction == eFree.
    Direction Trace Alignment axis mode: eFree, eCustom, or others. If eFree, trace is skipped.
    DirectionVector Trace Custom trace direction (used only with eCustom).
    Channel Trace Collision channel used for the trace (e.g., Visibility, WorldStatic).
    bComplex Trace If true, uses complex collision (triangle mesh).
    bIgnoreSelf Trace Ignores the actor that owns the trace component.
    ActorsToIgnore Actors List of additional actors to ignore during trace.
    bAlignNormal Align If true, aligns the instance with the hit surface's normal.
    AlignNormalMask Align Vector mask used to limit normal alignment to specific axes (e.g., Z only).
    bUseTraceDirection Align If true, uses the trace direction instead of the normal for alignment.
    TraceOffset Align Additional offset from hit location. Useful for offsetting above the surface.
    bPostHitApplyLocation Align Applies final location after trace hit.
    bPostHitApplyRotation Align Applies final rotation after trace hit.
    bDebug Debug If true, enables drawing of debug line/traces in editor/game.
